# PixivDL Browser v0.5.1

PixivDL Browser v0.5.1 是在 `0.5.0` 基础上的整理发布。它继续保持无本地后端、无 Cookie 导入的浏览器扩展形态，同时把当前已验证的工作台布局、保存目录显示、收藏仓库操作和协作说明同步进仓库与发布包。

## 适合谁

- 已经在 Chrome 或 Edge 中登录 Pixiv 的用户。
- 想通过 PID 快速查看并保存单个 Pixiv 作品的用户。
- 需要把下载保存到浏览器下载目录下指定子目录，并希望界面明确显示实际保存位置的用户。
- 需要在当前浏览器配置文件内启用本地 HTTP/HTTPS/SOCKS 代理的用户。
- 希望维护一个可搜索、可导入导出、可直接下载作品的本地收藏仓库的用户。
- 需要在多窗口或多代理协作下继续维护这个仓库的用户。

## 主要功能

- 按 PID 获取 Pixiv 插画/漫画作品信息。
- 弹窗按作品预览、图片选择、浏览器下载三栏组织，收藏状态、已选数量和下载进度会即时反馈。
- 支持预览、选择页面、全选/取消全选、ZIP 下载和逐张原图下载。
- 支持从弹窗打开完整工作台，新标签页会使用更宽的三栏布局，并可带入当前 PID。
- 保存目录以 `浏览器下载目录\PixivDL` 这样的有效位置显示，实际仍按浏览器下载 API 要求作为相对路径处理。
- “浏览”按钮会打开浏览器默认下载目录，保存子目录会在下载时由浏览器创建。
- 新增本地代理设置，支持 HTTP、HTTPS、SOCKS4 和 SOCKS5。
- 代理设置通过 `chrome.proxy.settings` 作用于当前浏览器配置文件，不修改 Windows 系统代理。
- 收藏仓库支持弹窗内搜索，以及新标签页图库、固定搜索框、右侧详情面板、导入/导出 JSON。
- 收藏卡片和详情面板都可以发起下载，下载前可选择 ZIP 或文件模式。
- 重复收藏会被跳过，导入收藏时也会跳过重复 PID。
- Pixiv 请求失败时，空状态区提供原 Pixiv 作品页入口。
- 文档统一改为推荐使用稳定目录 `PixivDL-Browser-latest` 进行升级和数据保留。
- 仓库内补充了协作文档和项目本地 skill，便于后续多窗口继续维护。

## 安装

1. 下载 `PixivDL-Browser-0.5.1.zip`。
2. 解压 ZIP 文件。
3. 为了保留收藏和缓存，建议把内容覆盖到固定目录，例如 `PixivDL-Browser-latest`。
4. 打开 Chrome 或 Edge 的扩展管理页。
5. 开启“开发者模式”。
6. 点击“加载已解压的扩展程序”，选择固定目录。
7. 后续升级时继续覆盖同一目录，并在扩展管理页点击刷新。

## 限制

- 只支持 Pixiv 静态插画和漫画图片，不支持 ugoira 动图。
- Pixiv 网页 AJAX 接口不是公开稳定 API，Pixiv 改站后可能需要维护。
- 保存目录是浏览器下载目录下的相对子目录，不是任意 Windows 绝对路径；扩展不能读取系统下载目录的真实绝对路径，也不提供任意文件夹写入。
- 代理设置属于浏览器扩展作用域；如果浏览器或策略禁止扩展代理，保存设置时会提示失败。
- 只应下载当前 Pixiv 账号有权限访问的内容，并遵守 Pixiv 及作者的使用规则。
